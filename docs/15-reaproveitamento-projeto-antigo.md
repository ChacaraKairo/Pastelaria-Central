# 15. Reaproveitamento do Projeto Antigo

## Repositório de referência

PI-Pastelaria-2024-01.

## Objetivo

Analisar o que pode ser reaproveitado do projeto antigo feito para pastelaria.

## Muito aproveitável

- Estrutura de backend.
- Rotas.
- Controllers.
- Models.
- Ideia de pedidos.
- Itens de pedido.
- Usuários.
- Cozinha.
- Configuração com Node.js e Express.

## Aproveitável com adaptação

- Comandas.
- Mesas.
- Relatórios.
- Fluxo interno de atendimento.

## Pouco útil para o MVP 1

- Sistema completo de comandas.
- Regras presenciais complexas.
- Relatórios avançados.
- Swagger, caso atrase o desenvolvimento inicial.

## Como será usado

No MVP 1, o projeto antigo servirá como referência conceitual.

No MVP 2, ele poderá servir como base para:

- backend;
- painel administrativo;
- pedidos;
- cozinha;
- controle de status;
- cadastro de produtos.

## Decisão

Não copiar o projeto inteiro. Reaproveitar ideias, estrutura e partes úteis quando o backend for iniciado.
