# 20. Critérios de Aceite e Testes

## Objetivo

Definir como validar se o MVP está funcionando.

## Testes da página inicial

- A home carrega corretamente.
- Os botões principais aparecem no celular.
- O botão de pedido leva ao cardápio ou carrinho.
- O botão de encomenda leva ao formulário correto.

## Testes do cardápio

- Produtos aparecem com nome, descrição, preço e imagem.
- Categorias filtram corretamente.
- Produto indisponível não pode ser adicionado.

## Testes do carrinho

- Produto é adicionado corretamente.
- Quantidade pode ser alterada.
- Produto pode ser removido.
- Total é calculado corretamente.

## Testes do pedido via WhatsApp

- Campos obrigatórios são validados.
- Endereço é exigido quando o cliente escolhe entrega.
- Mensagem do WhatsApp é gerada corretamente.
- Link abre conversa com o número configurado.

## Testes da encomenda

- Campos obrigatórios são validados.
- Mensagem de encomenda é gerada corretamente.
- A mensagem deixa claro que é solicitação de orçamento.

## Testes de responsividade

- Site funciona em celular.
- Site funciona em desktop.
- Botões são fáceis de clicar.
- Carrinho não quebra em telas pequenas.

## Critério final de aceite

O MVP será aceito quando um cliente conseguir fazer um pedido e uma solicitação de encomenda pelo celular, enviando todas as informações corretamente pelo WhatsApp.
